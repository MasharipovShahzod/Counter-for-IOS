## Обзор проекта

Приложение использует камеру iPhone для захвата скелета пользователя через фреймворк **Vision** от Apple. Алгоритм конечного автомата анализирует углы суставов в реальном времени и засчитывает только технически правильно выполненные повторения. Все результаты тренировки шифруются и отправляются на сервер верификации, что исключает любую возможность мошенничества.

## Архитектурный стек

| Слой | Технология | Назначение |
|---|---|---|
| **UI** | SwiftUI | Декларативный интерфейс с реактивным обновлением |
| **Компьютерное зрение** | Vision Framework | Обнаружение и трекинг 17 точек скелета |
| **Машинное обучение** | CoreML | Ускорение инференса на Neural Engine (ANE) |
| **Криптография** | CryptoKit | AES-256-GCM + ECDH (P-256) + HKDF-SHA256 |
| **Камера** | AVFoundation | Захват видеопотока с управлением сессией |
| **Конкурентность** | Swift Concurrency (async/await) | Обработка кадров без блокировки UI |

## Ключевые возможности

### 🤖 Автоматический подсчёт повторений
- Детекция скелета в реальном времени (17 ключевых точек тела)
- Поддержка двух упражнений: **отжимания** и **приседания**
- Конечный автомат с состояниями: `ready → descending → atBottom → ascending → ready`
- Адаптивный выбор стороны тела (левая / правая) по уровню уверенности Vision

### 📐 Сглаживание скелета через EMA-фильтр
- Экспоненциальное скользящее среднее (α = 0.6) для угловых значений суставов
- Устраняет дрожание суставов между кадрами без внесения задержки
- Настраиваемый порог минимальной уверенности (`minimumJointConfidence = 0.3`)

### 🛡 4-уровневый клиент-серверный античит

| Уровень | Название | Метод защиты |
|---|---|---|
| **L1** | Обнаружение живости | Дисперсия Z-прокси, микротремор, FNV-хэш кадра |
| **L2** | Целостность процесса | Джейлбрейк, Frida, P_TRACED, XOR-счётчик |
| **L3** | Криптореестр | AES-256-GCM + ECDH + SHA-256 цепочка хэшей |
| **L4** | Защита транспорта | SPKI-пиннинг TLS, Team ID, profiling profile |

### 🎙 Голосовая обратная связь
- Аудио-подсказки при неправильной технике (AVSpeechSynthesizer)
- Подавление дублирующихся сообщений (cooldown 2.5 сек)

## Установка и запуск

### Предварительные требования

- **macOS** 13.0 Ventura или новее
- **Xcode** 15.0 или новее
- **XcodeGen** (для генерации `.xcodeproj` из `project.yml`)

### Шаг 1 — Установка XcodeGen

```bash
brew install xcodegen
```

### Шаг 2 — Клонирование репозитория

```bash
git clone https://github.com/MasharipovShahzod/Counter-for-IOS.git
cd Counter-for-IOS
```

### Шаг 3 — Генерация Xcode-проекта

```bash
xcodegen generate
```

Команда создаёт `FitnessTracker.xcodeproj` из файла `project.yml`.

### Шаг 4 — Открытие в Xcode

```bash
open FitnessTracker.xcodeproj
```

### Шаг 5 — Настройка подписи

1. В Xcode откройте **Target → Signing & Capabilities**
2. Выберите вашу **Team** и настройте **Bundle Identifier**
3. Подключите физическое устройство (Vision Framework не поддерживается в симуляторе)

### Шаг 6 — Запуск

Нажмите **⌘R** или кнопку **Run** в Xcode.

> ⚠️ Для работы Vision требуется реальное устройство с **A12 Bionic** или новее (iPhone XS / 2018+).

## Система безопасности

> **Текущий статус: слои 1–2 активны, слои 3–4 выключены.**
>
> Приложение подключает защиту само — см. `WorkoutViewModel.startSecurity()` и
> `makeSecurityConfiguration()`, где перечислено, что включено и почему.
> Криптореестр (L3) и SSL-пиннинг (L4) выключены, потому что в
> `SecurityConfiguration` пока стоят заглушки: хост `api.yourfitnessapp.com`,
> `expectedTeamID = "ВАШЕ_TEAM_ID"`. `softMode` включён по той же причине.
>
> **Без серверной верификации счётчик остаётся словом клиента.** Слои 1–2
> поднимают цену атаки, но не заменяют проверку на бэкенде. Порядок включения
> расписан в доке к `makeSecurityConfiguration()`.

### Использование `SecureWorkoutSession` напрямую

Экран тренировки делает это сам; ручная сборка нужна, только если вы встраиваете
трекер в свой UI.

```swift
// 1. Создайте менеджер и сессию
let tracker = ExerciseTrackerManager(exercise: .pushUp)
let security = SecureWorkoutSession(configuration: SecurityConfiguration())

// 2. Запустите сессию (ECDH handshake + предстартовые проверки)
try await security.startSession()

// 3. Подключайте к трекеру ТОЛЬКО после успешного старта: пока сессия неактивна,
//    validateFrame отклоняет каждый кадр, и трекер читает это как потерю трекинга.
tracker.secureSession = security

// 4. Тренировка идёт автоматически через делегат

// 5. Завершите и получите квитанцию сервера
let receipt = try await security.finishSession()
print("Верифицировано повторений: \(receipt.verifiedRepCount)")
```

### `ObfuscatedCounter` — защита счётчика в памяти

Счётчик повторений хранится в форме `maskedValue = realValue XOR salt XOR canary`, где `salt` и `canary` — криптографически случайные `UInt64`, перегенерируемые после каждого доступа. Это делает «заморозку» числа в GameGem / iGameGuardian невозможной.

## Серверная верификация

Бэкенд реализован на **FastAPI** (Python) и доступен в директории `backend/`.

### Запуск сервера

```bash
pip install fastapi uvicorn cryptography pydantic
uvicorn backend.workout_verification:app --reload --port 8080
```

### API эндпоинты

| Метод | URL | Назначение |
|---|---|---|
| `POST` | `/v1/session/init` | ECDH-рукопожатие, выдача сессионного ключа |
| `POST` | `/v1/workout/verify` | Расшифровка AES-GCM, верификация реестра |

Подробная спецификация API — в файле [Handoff.md](Handoff.md).
